## 3. Архитектура системы

### 3.1. Общая архитектура

Система построена по клиент-серверной архитектуре. Клиентская часть
реализована на фреймворке React, серверная — на FastAPI (Python).

### 3.3. Описание взаимодействия

1. Пользователь через браузер отправляет HTTP-запрос к серверу
2. Запрос проходит аутентификацию (JWT-токен)
3. Маршрутизатор (Router) направляет запрос в соответствующий сервис
4. Сервис выполняет бизнес-логику и обращается к базе данных
5. Результат возвращается клиенту в формате JSON
