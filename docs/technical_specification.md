# Техническое задание

## 1. Общие сведения

| Параметр | Значение |
|----------|----------|
| **Наименование системы** | «Электронный журнал учёта аварийных ситуаций» |
| **Назначение** | Автоматизация учёта и обработки аварийных ситуаций |
| **Тип системы** | Веб-приложение |

## 2. Функциональные требования

### Модуль 1: Аутентификация и авторизация
- Вход по логину и паролю
- Разграничение доступа по ролям
- Выход из учётной записи

### Модуль 2: Управление аварийными ситуациями
- Регистрация аварийной ситуации
- Редактирование
- Назначение ответственного
- Отметка о выполнении

### Модуль 3: Отчёты
- Отчёт по авариям за период
- Отчёт по ответственному сотруднику
- Статистика по типам аварий

## 3. Технологический стек

- **Backend:** Python + Django
- **Frontend:** React
- **База данных:** PostgreSQL
